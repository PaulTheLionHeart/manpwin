Hailstone Sequence

Visualisation of a 2D Collatz (hailstone) sequence.

Overview

The Hailstone Sequence in ManpWIN is a discrete dynamical system operating on integer coordinates (Z × Z). Starting from an initial point, transformation rules based on parity (even or odd values) generate a sequence of points that either form cycles or diverge outward.

Unlike traditional fractals based on complex numbers, the Hailstone sequence explores behaviour on an integer lattice, revealing surprising structure from simple rules.

Access

Fractal Type: Hailstone

Configuration

Starting Point

  • Start X – initial X coordinate
  • Start Y – initial Y coordinate
  • Default: (-10, 6)

Iterations

  • Max Iterations – limits sequence length

Display Options

  • Show Axes
  • Show Point Labels
  • Show Dots

Transformation Rules

The default 2D Hailstone system applies the following rules:

(x even, y even) -> (x/2, y/2)
(x even, y odd)  -> (x/2 + 1, 3y - 1)
(x odd, y even)  -> (3x ? 1, y/2 - 1)
(x odd, y odd)   -> (3x + 1, 3y - 3)

Visualisation

The display shows:

  • Sequence path as connected line segments
  • Cycle detection highlighted in magenta
  • Optional coordinate axes and tick marks
  • Point labels showing (N, X, Y)

Behaviour

If a sequence remains bounded, it must eventually enter a repeating cycle, as there are only a finite number of integer points within any bounded region.

  • Cycling – returns to a previous point and repeats
  • Divergence – grows without bound

If a sequence remains bounded, it must eventually cycle due to the finite number of integer points within any region.

Examples

  • (-10, 6) – complex behaviour before cycling
  • (1, 1) – rapid cycle
  • (-20, 15) – divergence
  • (0, 0) – origin behaviour
