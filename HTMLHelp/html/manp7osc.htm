Oscillators, Attractors and Dynamic Systems

Overview

Dynamic systems may be continuous (oscillators) or discrete (maps). These systems model physical and biological behaviour, although the images generated do not directly correspond to physical objects.

This section is divided into:

  • Suggestions and Procedure
  • Explanations and Justification

Using Oscillators

Select: Fractals -> Fractal Type -> Marcus Contributions -> Attractors from Oscillators.

Choose an oscillator (e.g. Aizawa) or any of the 1000+ available systems. Naming follows published papers where possible.

Morphing

Press Shift + M to animate transitions between attractors. Speed is controlled using + / -.

If animation appears jerky, increase the number of frames.

Axes Display

Displaying all axis pairs allows viewing every 2D projection. Click any image to expand.

Explanations and Justification

Number of Variables

ManpWIN doubles variables by storing past values using time step dt. This increases dimensionality without additional differential equations.

Morphing

All attractors contain the same number of points, allowing smooth linear transitions. Intermediate frames are valid attractors.

Coordinate Systems

The developers of ManpWIN accidentally overlooked that an oscillator was described by (rho, theta, phi) and plotted it as (x, y, z). The images generated did not match the published images but were nevertheless beautiful. Returning the program to use the oscillator as (rho, theta, phi) we returned to the original images of the paper. That was “the seed” to implement other 11 coordinate systems for all oscillators in ManpWIN. It allows all operations on those mathematical attractors as if they were real-world attractors.
