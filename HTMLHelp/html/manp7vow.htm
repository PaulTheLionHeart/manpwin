Fractal Types Supported

Fractals are selected from the Fractals Menu or by pressing ‘T’.

A Note about Multi-Threading:

Multi-threading can greatly improve calculation speed as it breaks the image up into columns and can use different processor cores to do the calculations in parallel. At this stage, multi-threading is only available on perturbation and slope fractals. This will improve as the legacy code is upgraded.

When the multi-thread setup dialogue box is selected, a number of delays are offered. These should not need to be changed, however, on rare occasions, a crash may occur and these can be increased. This will slow down the calculations. Internal perturbation delay greater than zero will cause the screen to be updated continuously.

As the screen memory is accessed by different threads at the same time, sometimes there is a clash and pixels may be coloured incorrectly. Using a Mutex to block simultaneous access, this can be overcome at the cost of speed.

Specific ManpWIN fractals:

Mandelbrot The original fractal. Refer to Scientific American August 1985. It has the basic driving equation Zn+1 = Zn^2 + C where  Zn+1 in the next value of Z, Zn is the current value and C is the position on the screen. Z is a complex number of the form A+iB where ‘i’ is the square root of -1.

Pertubation:

Recently a new method was developed to render Mandelbrot fractal images from a reference point rather than calculating the escape-speed for each pixel. Since lower precision then can be used, the time to render images decreases significantly. Compared with the fastest programs using the old method, this program is more than 100 times faster on zoom levels around e100! For details see sft_maths.pdf.

 

Most of the Mandelbrot derivatives described below are included as perturbation fractals. Some filters, including all the Tierazon filters and Tierazon Colour schemes, are also provided for perturbation fractals.

 

Perturbation fractals now allow slope calculations. There are two methods offered, analytic derivative and forward differencing. Only Mandelbrot and power offer derivative method as other fractals offered do not have continuous derivatives. Slope is discussed later in this section.

Burning Ship:

The Burning Ship fractal, first described and created by Michael Michelitsch and Otto E. Rössler in 1992, is generated by iterating the function:

 

      zi = abs(zr * zi) * 2.0 - JuliaI;
      zr = zrsqr – zisqr + JuliaR;

 

in the complex plane which will either escape or remain bounded. The difference between this calculation and that for the Mandelbrot set is that the real and imaginary components are set to their respective absolute values before squaring at each iteration. The mapping is non-analytic because its real and imaginary parts do not obey the Cauchy–Riemann equations.

 

Burning Ship N > 2:

The Burning Ship fractal is also implemented for powers greater than 2:

 

The function that is iterated to generate this fractal can be generalised to any arbitrary power:

            z = [|Re(z)| + i |Im(z)|]^N + c

As with the Mandelbrot set, this creates shapes that resemble the original shape but have increasingly many attached arms or buds. With N=2 we get the Burning Ship. The N=3 version has been called the "Bird of Prey."

 

The odd powers are symmetric around an axes that are oriented at +/- 45 degrees.  This is probably due to the symmetry of the absolute value functions. This property has been mentioned at this site (external link) back in 1999.

 

These images are all centred at (0,0) in the complex number plane, and all have a size of 5 horizontally and 4 vertically. This size is a bit too big and results in a lot of blank space in the images, but the N=2 fractal is offset from the origin, so the larger size is needed to have all the images drawn with a consistent location and size so they can be compared accurately.

Perpendicular Mandelbrot:

The equation is very similar to burning ship (see above) except one “absolutes” just the real factor when calculating the imaginary side.

      zi = abs(zr) * zi * -2.0 + JuliaI;
      zr = zrsqr – zisqr + JuliaR;

 

Perpendicular Burning Ship:

The equation is very similar to burning ship except you absolute just the imaginary factor instead of the real factor when calculating the imaginary side.

      zi = zr * abs(zi) * -2.0 + JuliaI;
      zr = zrsqr – zisqr + JuliaR;

 

Buffalo:

A derivative of the Burning Ship, the Buffalo fractal was composed as follows:

Let z' = |Re(zn)| + i |Im(zn)|

zn+1 = z'2 - z' + c

 

Cubic Buffalo - Z = abs(Z^3) +C, The abs() function is applied separately to both the real and imaginary portions of Z^3.

 

Note that any integer power of Z may be used. Only powers of 2 and 3 have been tested against the Kalles Frakteler program.

 

Perpendicular Buffalo:

      zi = zr * abs(zi) * -2.0 + JuliaI;

5th Order ABS Mandelbrot Variations (Complete Set by stardust4ever):

Stardust4ever gives this following description:

5th Order Mandelbrot Variations. These formula share remarkable similarities to my previous 3rd order formulas. The 5th Order polynomials each have six terms. Because i^2 is in the real domain, every even term is real and every odd term is imaginary. When Zi and Zr are factored out, that leaves all forth power terms remaining. To optimize the code for computational efficiency, additional variables are created for (Zr^2)^2, (Zi^2)^2 and (Zi^2)*(Zr^2). Each of these terms appears twice in the equation, reducing the number of necessary bignum multiplication operations significantly. It may be possible to further optimize code in software but this is beyond my scope.

            Note that for each one of these:

        zisqr = zi * zi;
        zrsqr = zr * zr;
        zisqrsqr = zisqr * zisqr;
        zrsqrsqr = zrsqr * zrsqr;
        zrzisqr = zrsqr * zisqr;

Direct fractal formula parser:

Any function that can be expressed in the form Fn(z)+c can be typed into ManpWIN and the fractal function interpreted. This is based on the Fractint formula parser but does not need a file to be read. The fields are loaded with a double click. A single click will run the currently loaded formula. Julia and Real Time Julia do not work on direct formula parser because one needs to change the initial conditions to create a Julia set.

Documentation is supplied in Formula documentation by Mark Peterson.

Slope fractals:

These methods give a 3D look to fractals. The slope is calculated and a light source illuminates different parts of the fractal differently. There are two methods offered: Analytic derivative and forward differencing.

Here are some advantages of each method of calculating a slope.

Analytic derivative:

·       more accurate

·       more efficient computation

 

Forward differencing:

·       works for formulas with no analytic derivative (eg any formula with abs)

·       works for any measure

·       can include textures, field lines, or anything else that colouring might use

·       can use different measures in exterior, interior, in set, orbit trap regions, etc.

 

Both methods are also offered in perturbation fractals. However, only Mandlebrot set and power fractals are available for the analytical derivative method because they have continuous derivatives.

 

ManpWIN now offers forward differencing on many of the raster based fractals. However, these are still slow as they need to calculate every pixel and, as yet, are not multi-threading. Not all give accurate slope calculations as there are difficulties in getting the best continuous iteration count for some fractals. Refer Fractal Options

 

There is some experimental algorithm to allow a moving palette in forward differencing. Set Palette Shift (Fwd Diff) to a small integer greater than 0.
