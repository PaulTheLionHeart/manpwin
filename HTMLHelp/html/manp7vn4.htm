Parameter Files and Command Line Options

Overview

Parameter files are used to save favourite locations and settings for fractals. They are compact and allow quick restoration of complex views.

General Parameters

-cScreen dimensions: Xleft, Ytop, Height
-mScreen processing method:
1 = 1 pass
2 = 2 pass
G = Guessing + passes
B = Boundary Tracing
T = Tesseral (tiling method)
H = Horizontal Blinds
V = Vertical Blinds
-jJulia set flag followed by x,y location
-tMaximum iterations (historically called "threshold") (1 - 50,000,000)
-iInside colour (0 ñ 255)
-33D plotting parameters:
Scaling X, Y, Z
Rotation X, Y, Z (degrees)
-uTrue colour palette:
Start RGB values
RGB increment values
-sSave image (PNG format)
-oRotation parameter (context-dependent)
-eExit after completion
-pPeriodicity
-rAutoStereo depth
-bBiomorph colour
-kDecomposition colours (power of 2)
-lLogarithmic palette (with start value)
-aPalette filename

Extended Tokens

Additional single-character tokens are used internally and in parameter files to control specialised behaviour.

-~Colour smoothing
-^Palette shift
-@Iteration scaling (divide iterations)
-#Lyapunov sequence
-$Distance estimation
-23D oscillator rotation
-DFilter method
-FFractal type
-GScreen size
-HStereo pairs
-NFunction selection
-QCoordinate system
-VBailout limit
-WParameter list
-XInvert fractal
-YPalette starting offset
-ZParameter requirements

These tokens are specified after the - prefix in parameter files.

Some of these tokens originate from early versions of ManpWIN and Fractint, and remain for compatibility with existing parameter files.

Fractal Type Encoding

Fractal types in ManpWIN are encoded using a numeric identifier followed by optional parameters. These identifiers correspond to internal fractal definitions.

Format

-f<type>[parameters]

Examples: 

-f004        Mandelbrot
-f0503       Power fractal (power = 3)
-f049B5      Cubic fractal (subtype B, colour option 5)

Parameters

Additional values may follow the fractal type depending on the selected fractal. These may include:

  • Subtype ñ variant of the fractal
  • Degree / Power ñ exponent used in iteration
  • Special ñ colouring or behavioural modifier
  • Additional parameters ñ context-specific values

Notes

  • The available fractal types are extensive and continue to evolve.
  • Some types originate from Fractint and legacy implementations.
  • Others are specific to ManpWIN (e.g. Oscillators, Maps, Hailstone).
  • Certain fractals use subtype lookup tables or named variants.

These codes can be determined by selecting a fractal and saving a parameter file from within ManpWIN.

Common Examples

050Power fractals
078Newton fractals

Palette Definition

Parameter files may include a palette definition used for colouring the fractal. This is stored as a compact encoded string following the Palette= keyword.

Format

Palette=
<encoded palette data>

The palette data is stored as a continuous encoded sequence representing colour values. This format is optimised for compact storage rather than readability.

Notes

  • The palette is automatically interpreted by ManpWIN when loading the parameter file.
  • It defines either palette-based or true-colour gradients depending on context.
  • The encoded format is not intended for manual editing.

Example

Palette=
0000c2000WA0cYA0000ec20e0G5ecYAeKH5LwJ5LKlFLwnFLKH5zwJ5zKlFzwnFz...

(The full palette string may be several lines long.)

Palette data is generated internally or via the palette editor and is typically not modified directly by users.
