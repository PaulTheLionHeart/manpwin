Precision and Speed

Arithmetic types are used in ManpWIN to balance performance and deep zoom precision.

Overview

ManpWIN uses a range of arithmetic types to balance rendering speed and numerical precision. As zoom depth increases, higher precision arithmetic is required to maintain accuracy, although this comes at the cost of performance.

Different techniques, including perturbation and extended exponent arithmetic, allow ManpWIN to achieve deep zooms efficiently while minimising computational overhead.

Arithmetic Types

  • Double precision (64-bit floating point)
  • Double-double precision (~30 decimal digits)
  • Quad-double precision (~60 decimal digits)
  • Floatexp (extended exponent range)
  • Arbitrary precision (MPFR-based)

Double Precision

Standard double precision arithmetic is used up to magnifications of approximately 1013. This provides excellent performance as hardware floating point (FPU) instructions are used.

Beyond this level, precision becomes insufficient and higher precision methods are required.

Perturbation

Perturbation is a high-performance technique that allows deep zoom rendering using standard double precision arithmetic.

It works by computing a high-precision reference orbit and then calculating nearby pixels using relative differences. This avoids the need for high-precision arithmetic for every pixel.

In perturbation, the limiting factor is not numerical precision, but the available exponent range of double precision. This limits zoom depth to approximately 10-308 (minimum representable scale).

Double-Double and Quad-Double

Double-double and quad-double arithmetic extend precision to approximately 30 and 60 decimal digits respectively.

These provide significantly higher precision than standard double precision while remaining much faster than arbitrary precision arithmetic.

These methods are available in many fractals, including the formula parser.

Floatexp

Floatexp extends standard double precision by increasing the exponent range, while keeping the same mantissa precision.

It is used in conjunction with perturbation to allow significantly deeper zooms, beyond the normal limit of approximately 10-308.

Using floatexp, ManpWIN has achieved zoom levels on the order of 1020000.

Floatexp does not increase numerical precision and is not a replacement for arbitrary precision arithmetic. Its purpose is to extend the range of values that can be represented.

Arbitrary Precision

Arbitrary precision arithmetic (MPFR) is used when all other methods are insufficient.

This allows extremely deep zooms but at a significant performance cost. Maximum precision is typically limited to around 3600 bits.

Slope and Arithmetic

Slope calculations introduce additional numerical requirements depending on the method used.

  • Analytic derivative methods may require higher precision arithmetic
  • Forward differencing generally works with standard precision

When slope is used together with perturbation, the perturbation engine still operates using double precision (or floatexp for extended range).

In this case, floatexp is used to extend the zoom depth of perturbation, not specifically for slope calculations.

Description

ManpWIN dynamically selects arithmetic types to balance performance and precision, allowing efficient rendering across a wide range of zoom levels.

Details

Advanced techniques such as perturbation and extended exponent arithmetic enable deep zoom exploration without the constant need for arbitrary precision.

Notes

Higher precision improves accuracy but reduces performance. Choosing the appropriate arithmetic type is important for achieving optimal results.
