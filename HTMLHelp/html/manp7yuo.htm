Known Limitations

Overview

ManpWIN is a mature and evolving system. While most core functionality is stable, some features remain experimental or have limitations depending on the selected fractal type and rendering mode.

Compatibility

  • Not all FRACTINT parameter (.par) files are fully compatible. Certain features such as rotation and skew may not be interpreted.
  • Not all FRACTINT fractals are implemented within ManpWIN.
  • Some TieraZon filters and mapping methods may produce different results compared to the original TieraZon implementation.

Rendering & Features

  • Perturbation rendering does not support all filter and colouring methods.
  • Slope rendering is not available for all fractals or all rendering modes.
  • Some slope methods (particularly those involving trigonometric functions and high precision arithmetic) may fall back to slower but stable implementations.
  • Certain advanced combinations of features (e.g. high precision + slope + complex formulas) may result in reduced performance.

Multi-threading

  • Most pixel-based fractals support multi-threaded rendering, where the image is divided into regions and processed in parallel.
  • Advanced rendering methods such as perturbation and slope use different processing models. These may also be multi-threaded, but do not always follow the same pixel-based approach.
  • As a result, certain features (such as filters, colouring methods, or display options) may behave differently depending on the rendering mode.
  • Some specialised or non-raster fractals may still operate in single-threaded mode.
  • Perturbation rendering separates reference calculation from pixel evaluation, which can lead to different performance and feature characteristics compared to standard pixel-based rendering.
  • Traditional FRACTINT-style fractals use the plotting modes available through the Fractal Options menu.
  • Advanced rendering methods such as perturbation and slope use dedicated dialogue boxes for selecting rendering modes and options. These differ from the standard pixel-based plotting controls.

File Handling

  • Saved images include fractal parameters but do not store the full rendering state. Interrupted renders cannot be resumed.

Notes

  • The program continues to evolve, and many of these limitations are progressively being addressed as legacy code is modernised.

These limitations reflect the flexibility and experimental nature of advanced fractal exploration.
