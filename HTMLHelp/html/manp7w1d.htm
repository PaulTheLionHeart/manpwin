Animation Menu

Currently, animation runs of up to 100,000 frames can be made. Creating 1024 frames of 640*480 pixels requires I GByte of memory if all frames are loaded into memory. It is not recommended to load more than this number of frames into memory and it is recommended that creation of MPEG files be done by either saving individual frames as PNG files and writing a list of files to a *.lst file or writing the MPEG file directly. These are options for Fourier, Julia and zoom animations. The default is set to loading all frames into memory.

Note that the fractal that is to be animated must be loaded before animating. If the appropriate fractal is not loaded, the user is prompted to select the fractal required.

Zoom animation                                 (Ctrl+A)

This option creates a script file to generate a sequence of PNG images starting with a width of 4.0 zooming in to the final image. The centre of the screen remains the same. The threshold level can be varied from the starting frame to the final frame and changes geometrically.

The Palette Shift parameter allows the movement of the palette as the zoom proceeds. The colours will move inwards for positive values and out for negative values. The value of 0 inhibits the movement. A value of 3 gives a good movement if 1000 frames are being generated.

Setup Julia animation                        (key J)

Any location on the Mandelbrot set is a seed point that creates a unique Julia set. Three methods for creating a sequence of Julia sets are available: Linear, circle and cardioid. A “test” button allows the user to see the path through the Mandelbrot set that the Julia sets will follow. For example, this is the cardioid shape within the Mandelbrot set that creates wonderful Julia set animation.

Some great circle Julia animations can be made at [-1.0, 0 radius 0.25], [-0.125, 0.75 radius 0.1], [-1.309, 0 radius 0.0595], [0.281, 0.5313 radius 0.044]

These animations assume that the Mandelbrot set is being used. Other powers of Z^n can be used, but give different results. For example, a cubic and a quartic will give a different epicycloid than the cardioid generated for the Mandelbrot set. 

 

Setup Fourier animation                    (Shft+F)

Creates an animation of the rotating vectors representing the harmonics of a waveform.

Setup Malthus Rate animation         (key M)

Creates an animation by changing the rate parameter.

Setup Ikeda animation                      (Ctrl+I)

Creates an animation by changing the rate ‘b’ parameter.

Setup Inversion animation                (Shft+I)

Any location on the Mandelbrot set can be the centre of a radius that generates the inverse of a fractal. As in the Julia set animation, three methods for creating an animation sequence are available: Linear, circle and cardioid. A test button allows the user to see the path through the Mandelbrot set that the inversion centre will follow.

Setup Parameter animation              (key P)

Most fractals can be animated by selecting a particular parameter to vary between specific values. “Parameter number to animate” specifies the specific parameter to change starting at 0 and all the way to the maximum number of parameters for that fractal – 1. All the current values of each parameter is shown and may be modified. Once the parameter to be varied is specified, the start and end values for that parameter can be set. Allow parameter animations to oscillate back and forwards through the frames by checking the “Return” option.

Setup Chaotic Oscillator, Fractal Map, Surface, Knot or Curve animation          (Ctrl+O)

The user can choose one of two options, rotation animation or evolution animation.

 

The default is rotation animation which creates an animation through 360 degrees. The user may choose the axis of rotation. Note that more than one axis may be chosen simultaneously.  Other fractals, such as all the 3D Lorenz attractors, 3D Kamtorus, Pickover and Rossler may be rotated. Many of the remaining fractals generate a warning, "Fractal type is not a Chaotic Oscillator or a 3D Fractal Map! Do you wish to change the fractal type first?" If the user responds with “No”, then an attempt will still be made to rotate the fractal using colour information for the transformation.

 

The alternative allows the user to watch the Oscillator or Fractal Map to evolve. The value of iteration count is critical in getting the best range. Some experimentation may be required. Often it is best to start with a lower iteration count than the default .

Setup Morphing between pairs of axes in Chaotic Oscillator, Fractal Map, Knot or Curve animation        (Shft+M)

This is similar to other animations except that one can allow the animation to pause when a new pair of axes is reached. The delay can be set by the user. Note that chaotic oscillators and curves can have points far from the centre and causes morphing to go “off the scale”. Fractal Maps and knots are much better behaved. More investigation is required to get a better algorithm for oscillators and curves.

Stop animation Generation               (Shft+S) or (key ESC)

Commands while animation is running:

Once the animation is seen on the screen, a number of options are available:

Increase speed of the animation                   (key +)           

Decrease speed of the animation                 (key -)

Suspend/resume animation                           (S' key)

Escape from animation                                  (key ESC)

Single step through animation                      (space bar)

Resume single step                                        (‘S’ key twice)

 

These speeds are written to GIF files only and have no effect on MPEG frame rate.

 

Reverse direction of animation                    (key R)

Save all frames as PNG files                        (key A)           

Save frames in an animated GIF file            (key G)

Save frames in a MPEG-2 file                      (key M)

 

Note that the MPEG frame rate is set at 25 frames a second.

 

Escape from GIF and MPEG creation         (key ESC)
